## 노션너굴 업무관리 템플릿 핵심

상품 페이지와 블로그/영상 설명을 종합하면 이 템플릿의 핵심은 "프로젝트 중심 개인/업무 운영 시스템"이다.

### 정보 구조

- 프로젝트를 큰 목표 단위로 생성한다.
- 프로젝트는 업무 카테고리로 분류된다.
- 프로젝트 상태는 PARA에 가깝게 운영된다.
  - Projects: 현재 진행 중인 메인 작업
  - Areas: 상시 관리 영역
  - Resources: 자주 참고할 자료
  - Archives: 끝났거나 보관할 항목
- 프로젝트 내부에는 세부 할 일 DB가 있고, 상태 변경에 따라 진행률이 계산된다.
- 세부 할 일은 날짜 기반으로 프로젝트별 캘린더와 전체 대시보드 캘린더에 모인다.
- 오늘 할 일은 데일리 페이지에서 계획하고, 밀린 업무는 별도 탭에서 다시 끌어올 수 있다.
- 자주 참고할 문서나 양식은 핀으로 고정한다.
- 하루 마감에는 감정/일기/완료 상태를 기록하고, 감정은 그래프화된다.

### 사용성 원칙

- 매일 기록하지 못해도 시스템이 무너지지 않도록 설계한다.
- 빈 캔버스보다 정해진 시작 구조를 제공한다.
- 사용자는 원본 DB를 직접 수정해 습관, 카테고리, 상태를 바꾼다.
- 보드, 캘린더, 탭 필터를 조합해 같은 데이터를 다른 관점으로 본다.
- "밀린 업무"처럼 실패/누락을 숨기지 않고 다시 회수하는 뷰를 둔다.

### 자동화 패턴

- Notion Calendar와 연동해 프로젝트/세부 업무/오늘 할 일을 시간 기반으로 조정한다.
- Google Calendar로 큰 계획을 세우고, Make 자동화로 매일 새벽 Notion 오늘 할 일 DB에 일정이 들어오게 한다.
- 계획과 실행을 나눠 기록해 나중에 비교할 수 있게 한다.
- 2026-04-19 업데이트에서는 Claude Code를 활용한 AI 분석 리포트가 추가되었다.

우리 제품에 적용할 점:

- 프로젝트/문서/일감은 하나의 데이터 원본을 공유하고, 보드/캘린더/타임라인/필터만 바꿔 보여준다.
- "완료되지 않은 것"을 별도로 회수하는 뷰가 있어야 누락 방지가 된다.
- AI 리포트는 단순 요약보다 계획 대비 실행 비교가 핵심이다.
- 사용자가 코드 없이 바꾸는 설정 파일 또는 프로젝트 설정 DB가 필요하다.

## 너굴맵 핵심

너굴맵은 Notion DB의 관계형/롤업 구조를 캔버스에서 시각화하고, Notion URL import 및 Notion 워크스페이스 배포를 제공한다.

### 랜딩에서 확인한 기능

- 공개 템플릿 11개 제공
- 관계형/롤업 연결을 색상으로 구분
- 통합 토큰으로 Notion 워크스페이스에 배포
- Notion에서 DB나 속성이 바뀌면 30초 안에 구조도 갱신
- 새 DB 자동 발견
- 카드 위치와 레이아웃 보존
- 토큰 AES-256-GCM 암호화 저장

### 앱 HTML에서 확인한 기능

- 로그인 후 사용자별 구조도 저장
- 내 구조도/공유 구조도 분리
- 새 구조도, Notion URL import, 휴지통, 복구
- 이미지 저장과 Notion 배포
- 편집자/뷰어 공유
- 구조도당 최대 5명 공유 정책
- 저장 충돌 감지와 "최신 버전 가져오기" 옵션
- 10분 간격 자동 백업과 최근 50개 버전 복구
- Free/Pro 플랜 구분
- Pro에서 Notion 자동 동기화와 공유 기능 제공

우리 제품에 적용할 점:

- Git repo import도 "URL + 권한 토큰 + import limit + sync status"로 다뤄야 한다.
- 공유 권한은 viewer/editor/admin으로 단순화한다.
- 충돌이 발생하면 조용히 덮어쓰지 말고 저장 충돌 모달과 최신 버전 가져오기를 제공한다.
- 시각 구조도는 Notion DB가 아니라 레포/문서/이슈/API 관계도에 적용할 수 있다.
- 토큰 저장은 암호화와 명시적 동의가 필요하다.

## weekly-review GitHub 예제

[notionneogul/weekly-review](https://github.com/notionneogul/weekly-review)는 Google Calendar 계획과 Notion 할 일을 비교해 Claude로 주간 리포트를 만들고 Notion DB에 업로드하는 Node.js 프로젝트다.

### 확인한 파일

- `package.json`: `@anthropic-ai/sdk`, `@notionhq/client`, `googleapis`, `dotenv` 사용
- `src/index.js`: 날짜 범위 계산, Google/Notion 데이터 병렬 수집, 일별 비교, 주간 집계, Claude 분석, Notion 업로드
- `src/comparator.js`: 제목 정확 매칭, 부분 매칭, 시간 겹침 매칭, 실행률/공백/점수 계산
- `src/analyzer.js`: 설정 파일을 반영해 Claude에게 JSON 분석 결과를 요청
- `src/notion-read.js`: Notion 할 일 DB를 날짜/상태 기준으로 조회
- `분석기준.txt`: 상태 분류, 상태 의미, 습관 목록, 시간대, 등급, 점수 가중치, 분석 관점, 분석 스타일을 비개발자도 수정 가능하게 둠

우리 제품에 적용할 점:

- AI 자동화 기준은 코드가 아니라 프로젝트 설정으로 분리한다.
- 계획 대비 실행 비교는 제목 매칭, 부분 매칭, 시간 겹침, 코드 변경 근거 매칭을 순서대로 수행한다.
- AI 결과는 JSON/structured output으로 받고, UI에서 승인 가능한 액션으로 변환한다.
- 리포트는 문서로 끝내지 말고, 이슈 상태 변경/다음 주 일정 제안/문서 최신화로 이어져야 한다.

## 종합 인사이트

1. Notion 템플릿의 강점은 자유도가 아니라 반복 가능한 정보 구조다.
2. 너굴맵의 강점은 복잡한 관계를 시각화하고 import/export/sync로 실제 시스템과 연결하는 점이다.
3. weekly-review의 강점은 계획과 실행을 비교해 AI가 구체적 리포트를 만드는 점이다.
4. 우리 협업툴의 차별점은 이 세 가지를 GitHub 개발 프로젝트에 맞게 결합하는 것이다.

따라서 제품은 다음 방향이 맞다.

- `Git repo`를 Notion DB 원본처럼 다룬다.
- `docs/`, `issues`, `projects`, `pull requests`, `source files`를 하나의 지식 그래프로 만든다.
- Markdown 문서는 frontmatter로 상태/담당자/일정/연결 이슈를 가진다.
- 같은 원본 데이터를 보드, 캘린더, 타임라인, 위키, 코드 뷰어, 관계도로 보여준다.
- AI는 RAG로 답하고, MCP/GitHub API로 실제 변경을 수행한다.
